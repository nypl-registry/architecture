Space/Time Directory Architecture

This diagram shows current state of the Space/Time Directory project technical architecture. The diagram shows how tools and components work together, and how they share data. It does not show the final state of the project â€” new blocks will be added as the project procedes and work is started on new datasets, data processing tools and user interfaces.

In this diagram, only code and data components of the Space/Time Directory project are shown. Other, and equally important parts of the project (e.g. outreach, meetups, hackathons, collaborations), are (currently) not shown in this diagram.

Finished components are green, components on which work has not started are white, and components on which we are currently working are shaded green according to their finished state. External, non-Space/Time components have a dashed line. For components with associated GitHub repositories, the number of open issues are shown in the red badges.

This architecture diagram can be found on GitHub too. Please open an issue if the diagram is incorrect or out of date.
